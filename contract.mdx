The `TradingArena` contract is deployed on **Avalanche Fuji Testnet** and handles all on-chain logic: arena creation, player deposits, virtual trading, winner selection, and reward payouts.

## Access control

| Role | Address | Permissions |
| --- | --- | --- |
| `owner` | Deployer wallet | `createArena`, `deleteArena`, `withdrawCommission`, `setBackend`, `transferOwnership` |
| `backend` | Backend server wallet | `startBattle`, `endBattle` |
| `agent` | Bot wallet | `executeOrder`, `closeOrder` |
| `player` | Human wallet | `joinArena`, `claimReward` |

<Warning>
  An address cannot be both a player and an agent simultaneously. The contract enforces strict role separation — registering as an agent blocks the same address from joining as a player, and vice versa.
</Warning>

## Owner functions

### `createArena`

```solidity
function createArena(ArenaConfig calldata config)
  external
  onlyOwner
  returns (uint256 arenaId)
```

Creates a new arena with the given configuration. Returns the assigned `arenaId`.

**`ArenaConfig` fields:**

| Field | Type | Description |
| --- | --- | --- |
| `entryDeposit` | `uint256` | Required AVAX deposit to enter (wei). Must be `> 0` |
| `maxParticipants` | `uint256` | Maximum players per battle. Must be `≥ 2` |
| `minParticipants` | `uint256` | Minimum players to start. Must be `≥ 2` and `≤ maxParticipants` |
| `duration` | `uint256` | Battle duration in seconds |
| `cooldown` | `uint256` | Minimum seconds between consecutive battles in the same arena |
| `commissionBps` | `uint16` | Owner commission in basis points. Max `2000` (20%) |

**Reverts:** `InvalidParticipants`, `InvalidCommission`, `InsufficientDeposit`

---

### `deleteArena`

```solidity
function deleteArena(uint256 arenaId)
  external
  onlyOwner
```

Deletes an arena permanently. All queued and idle players are automatically refunded via `pendingReward` (pull-pattern). Cannot be called while a battle is `Active`.

**Reverts:** `ArenaDoesNotExist`, `ArenaHasActiveBattle`

## Player functions

### `joinArena`

```solidity
function joinArena(
  uint256 arenaId,
  address agentAddress,
  string calldata agentName
) external payable
```

Player enters an arena by sending the required AVAX deposit. The `agentAddress` is the bot wallet that will trade on the player's behalf.

- If the arena is `Idle` and not full → player joins the current battle directly
- If the arena is `Active`, full, or `Finished` → player enters the **FIFO queue** for the next battle
- If `msg.value > entryDeposit` → excess AVAX is automatically refunded

**Reverts:** `InsufficientDeposit`, `ZeroAddress`, `OwnerCannotBeAgent`, `AgentCannotBeOwner`, `AgentAlreadyRegistered`, `AlreadyJoined`

---

### `claimReward`

```solidity
function claimReward()
  external
```

Withdraws the caller's pending reward balance. Callable by any player with a non-zero `pendingReward`.

**Reverts:** `NoPendingReward`, `TransferFailed`

## Backend functions

### `startBattle`

```solidity
function startBattle(uint256 arenaId)
  external
  onlyBackend
```

Starts the battle in the given arena. Pulls queued players into the battle to fill remaining spots before starting. Enforces the `cooldown` period between consecutive battles.

**Reverts:** `ArenaNotIdle`, `ArenaCooldownNotMet`, `NotEnoughParticipants`

---

### `endBattle`

```solidity
function endBattle(uint256 arenaId, uint256 calldata currentPrices)[1]
  external
  onlyBackend
```

Ends the active battle. Marks all open token holdings to market using `currentPrices`, calculates total PnL for each participant, selects the winner, and distributes the prize pool.

`currentPrices` must be ordered as `[avaxPrice, btcPrice, ethPrice]` in 18-decimal USDT values.

**Winner formula:**

```text
totalPnl = realizedPnl + virtualUSDT + openPositionsValue - VIRTUAL_STARTING_BALANCE
```

The participant with the highest `totalPnl` receives `totalDeposits - commission`.

**Reverts:** `ArenaNotActive`, `DurationNotElapsed`

## Agent trading functions

### `executeOrder`

```solidity
function executeOrder(
  uint256 arenaId,
  TokenId tokenId,
  uint256 price,
  uint256 usdtAmount
) external onlyAgent
```

Agent spends virtual USDT to buy a token. Multiple calls accumulate into a single holding using a **weighted average entry price**.

**Weighted average price formula:**

```text
newAvgPrice = (oldBalance × oldAvg + tokenAmount × price) / (oldBalance + tokenAmount)
```

**Reverts:** `ArenaNotActive`, `InsufficientVirtualBalance`

---

### `closeOrder`

```solidity
function closeOrder(
  uint256 arenaId,
  TokenId tokenId,
  uint256 closePrice,
  uint256 sellAmount
) external onlyAgent
```

Agent sells a token holding (partially or fully) back to virtual USDT. Pass `type(uint256).max` as `sellAmount` to close the entire position.

**PnL formula:**

```text
pnl = (closePrice − avgEntryPrice) × sellAmount / 1e18
```

**Reverts:** `ArenaNotActive`, `PositionNotOpen`

## Custom errors

| Error | Cause |
| --- | --- |
| `NotOwner()` | Caller is not the contract owner |
| `NotBackend()` | Caller is not the authorized backend wallet |
| `NotAgent()` | Caller is not a registered agent in this battle |
| `ArenaDoesNotExist()` | `arenaId` does not exist |
| `ArenaNotIdle()` | `startBattle` called when arena is not in `Idle` state |
| `ArenaNotActive()` | Trading or `endBattle` called when arena is not `Active` |
| `ArenaHasActiveBattle()` | `deleteArena` called while a battle is running |
| `ArenaCooldownNotMet(uint256 readyAt)` | `startBattle` called before cooldown expires. `readyAt` = earliest allowed timestamp |
| `DurationNotElapsed(uint256 endsAt)` | `endBattle` called before battle duration ends |
| `NotEnoughParticipants(uint256 current, uint256 required)` | Not enough players to start |
| `InsufficientDeposit(uint256 sent, uint256 required)` | `msg.value` too low |
| `InsufficientVirtualBalance(uint256 available, uint256 required)` | Agent has insufficient virtual USDT |
| `AgentAlreadyRegistered()` | `agentAddress` is already registered in another active battle |
| `OwnerCannotBeAgent()` | Player wallet used as agent address |
| `AgentCannotBeOwner()` | Agent wallet attempting to join as player |
| `AlreadyJoined()` | Player already active in this battle |
| `PositionNotOpen()` | `closeOrder` called with no open holding |
| `ZeroAddress()` | Address parameter is `address(0)` |
| `TransferFailed()` | Native AVAX transfer failed |
| `NoPendingReward()` | `claimReward` called with zero balance |
